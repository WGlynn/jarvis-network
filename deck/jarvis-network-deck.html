> mechanism

Five architectural moves.
All in code.

01
Shard by function
Each call's prompt scoped to intent; no monolithic context.
→ lean calls
02
Triage by cost
Haiku classifies every message; only ~15% hit the full model.
→ 85% saved
03
Ground in files
LLM queries archive via tools instead of re-ingesting history.
→ no stale context
04
Template deterministic output
Digests / stats render from aggregated data. Zero generated tokens.
→ 0 inference cost
05
Route across providers
Claude, OpenRouter, Ollama. Cheapest adequate per task.
→ no lock-in

> move 01 :: sharding

Two kinds of sharding.
No monolithic bot.

[ CHAT-LEVEL ]
Router → N workers by chat ID.
  • Each worker holds state only for its assigned chats.
  • TG updates dispatch to the right shard, no cross-talk.
  • Add a customer → no scale hit on existing communities.
[ FUNCTION-LEVEL ]
Tool bundles per domain.
  • trading · social · security · memehunter · portfolio
  • Workflow router picks the smallest relevant bundle per intent.
  • Smaller prompt = fewer tokens per call.
// No giant monolithic bot loading every conversation's state on every turn.
